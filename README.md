# Сервис хранения заметок (MVP) с BDD-тестами

Данный проект содержит минимально жизнеспособный продукт (MVP) API-сервиса для управления заметками, а также комплекс автоматизированных приёмочных тестов, написанных с использованием BDD-подхода.

## 🛠️ Примененные технологии и паттерны
- **Язык программирования:** Python
- **Фреймворк для API:** FastAPI / Uvicorn
- **BDD-библиотека:** Behave (синтаксис Gherkin)
- **Сетевой клиент:** Requests
- **Паттерн Page Object:** Изолирует логику отправки HTTP-запросов к API (`utils/page_object.py`).
- **Паттерн Factory:** Отвечает за генерацию валидных и дефектных тестовых данных (`utils/factory.py`).

## 📋 Покрытие тестами
Реализовано 10 сценариев (позитивные и негативные кейсы) для всех базовых операций:
1. Создание новой заметки (валидные/невалидные данные).
2. Получение списка всех заметок (когда база пуста / когда есть данные).
3. Получение отдельной заметки по ID (валидный/невалидный ID).
4. Редактирование заметки по ID (валидный/невалидный ID).
5. Удаление заметки по ID (валидный/невалидный ID).

## 🚀 Инструкция по запуску

1. Установите необходимые зависимости:
   ```bash
   pip install fastapi uvicorn behave requests
   ```

2. Запустите API-сервер заметок:
   ```bash
   python main.py
   ```

3. В отдельном окне терминала запустите автоматические BDD-тесты:
   ```bash
   python -m behave
   ```
